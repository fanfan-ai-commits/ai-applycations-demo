# 斗鸡博弈 (Chicken Game) - 技术规范

## 1. 项目概述

- **项目名称**: 斗鸡博弈 - 胆怯游戏
- **项目类型**: 网页小游戏
- **核心功能**: 玩家与AI进行5轮赛车博弈，通过选择"前进"或"转弯"来获得更高分数
- **目标用户**: 对博弈论感兴趣的大学生和游戏玩家

## 2. UI/UX 规范

### 2.1 布局结构


- **移动端**: < 640px (单列布局，垂直堆叠)

### 2.4 组件规范

#### 赛道区域

- 高度: 200px
- 背景: 渐变灰色赛道纹理
- 中线: 虚线分隔
- 赛车: 🚗 emoji 或 SVG 图标

#### 得分面板

- 圆角: 16px
- 背景: 白色 + 柔和阴影
- 数字: 大号字体，动态计数动画

#### 按钮

- **前进按钮**:
  - 背景: linear-gradient(135deg, #22C55E, #16A34A)
  - 悬停: scale(1.05), 增强阴影
  - 图标: 🚗
  - 文字: "前进" + 收益说明
  
- **转弯按钮**:
  - 背景: linear-gradient(135deg, #6366F1, #4F46E5)
  - 悬停: scale(1.05), 增强阴影
  - 图标: 🔙
  - 文字: "转弯" + 收益说明

- **禁用状态**: opacity: 0.5, cursor: not-allowed

#### 对话气泡

- AI消息: 靠左，浅蓝背景 (#EFF6FF)，左边框蓝色
- 系统消息: 居中，浅灰背景，无边框
- 动画: slideIn 从左侧滑入

#### 历史记录

- 每条记录: 时间戳 + 玩家选择 + AI选择 + 结果
- 颜色编码: 绿色=玩家赢，红色=AI赢，灰色=平局

### 2.5 动画效果

- **页面加载**: fadeIn 0.5s
- **赛车移动**: 0.8s ease-out + 弹性 bounce
- **撞击效果**: scale + shake 动画
- **得分变化**: 数字滚动动画
- **按钮悬停**: transform scale 0.2s
- **新消息**: slideInUp 0.3s

## 3. 功能规范

### 3.1 游戏规则

```
收益矩阵:
            AI前进      AI转弯
玩家前进    -100, -100   +10, -5
玩家转弯    -5, +10      0, 0

游戏流程:
1. 共5轮博弈
2. 每轮玩家先选择
3. 玩家选择后，AI也做出选择（隐藏）
4. 展示动画结果
5. 记录得分
6. 5轮后显示最终结果
```

### 3.2 AI 性格系统

| 性格 | 前进概率 | 描述 |
|------|----------|------|
| 激进型 | 80% | 喜欢冲锋，言语挑衅 |
| 保守型 | 30% | 谨慎驾驶，防守为主 |
| 随机型 | 50% | 不可预测 |
| 学习型 | 动态 | 根据玩家历史调整 |

- 开局让玩家选择AI性格
- AI根据性格决定每轮选择

### 3.3 AI 对话系统

#### 状态分类

1. **开局**: 挑衅、宣战
2. **玩家前进时**: 紧张/兴奋/应战
3. **玩家转弯时**: 嘲笑/得意
4. **撞车时**: 惊讶/后悔
5. **胜利时**: 嘲讽
6. **失败时**: 不服/威胁

#### 对话示例

### 3.4 数据处理

- **游戏状态**:
  - currentRound: 1-5
  - playerScore: 整数
  - aiScore: 整数
  - history: 数组，每轮 {round, playerChoice, aiChoice, playerDelta, aiDelta}
  - aiPersonality: 选择的性格

- **本地存储**:
  - totalGames: 总游戏数
  - playerWins: 玩家获胜场次
  - aiWins: AI获胜场次

### 3.5 边缘情况

- API调用失败: 显示默认消息，重试按钮
- 网络超时: 30秒超时，显示提示
- 最后一轮: 显示"游戏结束"而非"下一轮"
